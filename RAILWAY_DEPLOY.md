# Railway deployment guide

## Подготовка к деплою

### 1. Создание проекта на Railway
1. Зайдите на [railway.app](https://railway.app)
2. Войдите через GitHub
3. Нажмите "New Project" → "Deploy from GitHub repo"
4. Выберите репозиторий с ботом


### 3. Настройка cookies (опционально)
Для лучшей работы с YouTube:
1. Создайте файл `cookies.youtube.txt` локально
2. Добавьте его в репозиторий (или создайте через Railway файловую систему)

### 4. Деплой
1. Railway автоматически соберет Docker контейнер
2. Бот запустится и будет доступен 24/7
3. Логи можно смотреть в Railway Dashboard → Deployments → Logs

## Особенности Railway

### Лимиты
- **RAM**: 512MB (бесплатно), 1GB+ (платно)
- **CPU**: Shared (бесплатно), Dedicated (платно)
- **Хранилище**: Эфемерное (файлы удаляются при перезапуске)

### Рекомендации
- Используйте внешнее хранилище для cookies (если нужно)
- Настройте мониторинг через Railway Dashboard
- Рассмотрите платный план для стабильности

### Мониторинг
- Railway Dashboard → Metrics - статистика использования
- Logs - логи работы бота
- Deployments - история деплоев

## Troubleshooting

### Бот не отвечает
1. Проверьте логи в Railway Dashboard
2. Убедитесь, что переменные окружения установлены
3. Проверьте статус деплоя

### Ошибки скачивания
1. FFmpeg установлен в контейнере
2. Проверьте доступность внешних сервисов
3. Добавьте cookies для YouTube (если нужно)

### Производительность
- Для высокой нагрузки рассмотрите платный план
- Оптимизируйте размер Docker образа
- Используйте кэширование где возможно
